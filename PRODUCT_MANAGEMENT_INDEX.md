## 🎯 Cara Menggunakan Dokumentasi Ini

### Untuk Product Manager:
1. **Planning:**
   - Gunakan PRD untuk define requirements
   - Gunakan Backlog untuk prioritize features
   - Gunakan Tickets untuk create detailed stories
   
2. **Sprint Planning:**
   - Review Backlog untuk upcoming sprint
   - Pilih stories dari Development Tickets
   - Estimate dengan team
   
3. **Tracking:**
   - Update status di Development Tickets
   - Update velocity di Backlog
   - Review metrics weekly

### Untuk Scrum Master:
1. **Ceremonies:**
   - Sprint Planning: Gunakan Backlog & Tickets
   - Daily Standup: Reference Tickets untuk progress
   - Sprint Review: Demo based on Tickets acceptance criteria
   - Retrospective: Review velocity & team performance
   
2. **Tracking:**
   - Update sprint goals di Backlog
   - Track velocity
   - Remove blockers

### Untuk Developer:
1. **Development:**
   - Reference PRD untuk context & requirements
   - Follow Tickets untuk implementation details
   - Check acceptance criteria sebelum submit
   
2. **Questions:**
   - PRD untuk understand business logic
   - Tickets untuk technical specs
   - Backlog untuk priorities

### Untuk QA Engineer:
1. **Testing:**
   - Use acceptance criteria dari Tickets
   - Reference PRD untuk expected behavior
   - Create test cases based on user stories
   
2. **Bug Reporting:**
   - Add to Known Issues di Backlog
   - Create bug ticket di Tickets format
   - Prioritize dengan PM

### Untuk Stakeholder:
1. **Updates:**
   - Review Roadmap di Backlog
   - Check OKRs progress
   - Review release plans
   
2. **Feedback:**
   - Submit feature requests di Backlog
   - Vote pada existing requests
   - Participate in sprint reviews

### Untuk Team Baru:

1. **Hari 1: Pahami Produk**
   - Baca Product Vision di Backlog
   - Baca Executive Summary di PRD
   - Review User Personas di PRD

2. **Hari 2: Pahami Technical**
   - Review Technical Architecture di PRD
   - Review Data Model/ERD di PRD
   - Setup development environment (lihat README.md)

3. **Hari 3: Pahami Current Work**
   - Review current sprint di Backlog
   - Check assigned tickets di Development Tickets
   - Attend daily standup

4. **Minggu 1: Mulai Contribute**
   - Pick first ticket (pilih yang priority LOW first)
   - Read acceptance criteria carefully
   - Ask questions di standup
   - Submit first PR

## 🎓 Best Practices

### When Creating New Features:

1. **Start with WHY**
   - Add user story di Backlog dengan voting
   - Discuss dengan team & stakeholders
   - Validate with user research

2. **Document in PRD**
   - Add feature description
   - Define user stories
   - Set acceptance criteria
   - Update data model if needed

3. **Break Down to Tickets**
   - Create Epic di Development Tickets
   - Break into Stories
   - Detail Subtasks dengan estimasi
   - Identify dependencies

4. **Plan in Backlog**
   - Prioritize in backlog
   - Estimate story points
   - Assign to sprint
   - Track progress

5. **Execute**
   - Follow DoR before start
   - Develop according to specs
   - Follow DoD before close
   - Update documentation
